# 腎臓くん日記

透析患者の食事管理と血圧管理を、ご家庭で毎日記録するためのアプリです。医療判断ではなく、日々の記録・傾向確認・意識づけを目的にしています。

## 公開方式

GitHub Pagesで公開する前提です。

- `index.html`: GitHub Pagesのトップページ
- `manifest.webmanifest`: PWA設定
- `sw.js`: オフライン表示用サービスワーカー
- `icons/`: ホーム画面追加用アイコン

## 主な機能

- 妻が見る画面
- 食事管理
- 血圧管理
- 履歴・グラフ
- 設定
- Googleスプレッドシート同期
- 共有コード（手動移行用）
- PWA対応
- iPhone Safari / Edge対応
- ホーム画面追加対応
- オフライン時に最後に読み込んだ画面を表示

## 保存方式・同期

記録データはまずブラウザ内に保存されます。Googleスプレッドシート同期URLを設定すると、保存時にスプレッドシートへ同期し、アプリ起動時や妻が見る画面を開いた時に最新データを読み込みます。

共有コードは手動移行用として残していますが、通常運用では毎回作成する必要はありません。

## Googleスプレッドシート同期の初回設定

1. Googleスプレッドシートを1つ作成します。
2. 拡張機能 → Apps Scriptを開きます。
3. `google-sheets-sync-apps-script.js` の内容を貼り付けて保存します。
4. デプロイ → 新しいデプロイ → 種類「ウェブアプリ」を選びます。
5. 実行ユーザーは自分、アクセスできるユーザーは利用環境に合わせて設定します。
6. 発行されたWebアプリURLを、アプリの「設定」→「Googleスプレッドシート同期」へ入力します。
7. 最初の端末で「この端末のデータを初回同期」を押すと、既存のローカルデータをスプレッドシートへ移行できます。

妻のiPhoneでは同じWebアプリURLを設定して「スプレッドシートから読込」を押すと、同じ同期先を参照します。

## GitHub Pages

GitHub Pagesではリポジトリのルートにある `index.html` が公開トップページになります。Pages設定は `main` ブランチのルート公開を想定しています。
